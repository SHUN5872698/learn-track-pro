
## 指示

ユーザーから提示された「達成したい大きな目標（ゴール）」と、関連する「現状の分析や課題をまとめたドキュメント」をインプットとして、Notionのデータベースにインポートしやすい構造化されたマークダウン形式のタスクリストを生成してください。

## 生成ルール

1.  **階層構造:** 親子関係はインデントで表現する。
2.  **タスクID:** 各タスクには、階層がわかるような一意のID（例: `T1.1`, `T1.2.1`）を必ず付与する。
3.  **ステータス:** 各タスクの末尾に、初期状態として `[Status: Not Started]` を付与する。
4.  **依存関係（ブロッキング）:** あるタスクが別のタスクの完了を待つ必要がある場合、`[Blocks: #タスクID]` の形式で依存関係を明記する。
5.  **親アイテムへのリンク:** ドキュメントの先頭に、このタスクリストがどの大きな工程に属するかを示すリンク（例: `**親アイテム:** [プロジェクトロードマップ](../project-roadmap.md)`）を記載する。
6.  **論理的な分割:** タスクは「フェーズ」や「機能」といった論理的なまとまりでセクション分けする。

## 生成例

```markdown
# ゴール: 〇〇機能の実装

**親アイテム:** [プロジェクトロードマップ](../project-roadmap.md)

---

## フェーズ1: データベース設計

- [ ] **T1.1:** `items`テーブルのマイグレーション作成 [Status: Not Started]
    - [ ] **T1.1.1:** スキーマを定義する。
- [ ] **T1.2:** `Item`モデルの作成 [Status: Not Started] [Blocks: #T1.1]

## フェーズ2: API実装

- [ ] **T2.1:** `ItemController`の作成 [Status: Not Started] [Blocks: #T1.2]
```
