# Workflow: move-documents

このプロンプトは、指定されたファイルを適切なディレクトリに移動させる `workflow:move-documents` の実行をガイドします。

## パラメータ:
- `source_file_path`: 移動するファイルのパスを指定します。

## 自動的な移動先の決定

以下のルールに基づいて移動先を決定します。

1.  **ファイル内容の分析**: `source_file_path` の内容を読み込み、その性質を判断します。
    -   API仕様、エンドポイント、リクエスト/レスポンス例に関する内容 (`API`, `エンドポイント`, `リクエスト`, `レスポンス`) -> `apis/`
    -   システム設計、コンポーネント図に関する内容 (`設計`, `アーキテクチャ`, `コンポーネント`) -> `architectures/`
    -   プロジェクト固有の開発手順やレシピに関する内容 (`手順`, `レシピ`, `開発ガイド`) -> `cookbooks/`
    -   データベーススキーマ、ER図、マイグレーション戦略に関する内容 (`データベース`, `スキーマ`, `ER図`, `テーブル定義`) -> `databases/`
    -   設計上の意思決定の記録 (ADR) に関する内容 (`決定`, `ADR`, `意思決定`) -> `decisions/`
    -   UI/UXデザイン、ワイヤーフレーム、モックアップ、デザインシステムに関する内容 (`UI`, `UX`, `画面`, `デザイン`, `ワイヤーフレーム`, `モックアップ`) -> `designs/`
    -   開発ワークフロー、コーディング規約、一般的な開発プロセスチェックリストに関する内容 (`ワークフロー`, `コーディング規約`, `開発プロセス`) -> `development-processes/`
    -   デプロイ、サーバー設定、Docker、CI/CD、監視に関する内容 (`デプロイ`, `サーバー`, `Docker`, `CI/CD`, `監視`) -> `infrastructures/`
    -   プロジェクトロードマップ、一般的なプロジェクト情報、貢献ガイドに関する内容 (`ロードマップ`, `プロジェクト情報`, `貢献ガイド`) -> `project-infos/`
    -   認証・認可、CSRF/XSS対策、セキュリティベストプラクティスに関する内容 (`認証`, `認可`, `セキュリティ`, `CSRF`, `XSS`) -> `securities/`
    -   プロジェクト固有の具体的なタスクリスト、機能実装チェックリストに関する内容 (`タスク`, `チェックリスト`, `機能実装`) -> `tasks/`
    -   テスト戦略、単体/結合/E2Eテストガイドライン、テストレポートに関する内容 (`テスト`, `単体テスト`, `E2Eテスト`) -> `tests/`
2.  **デフォルトパス**: 上記のいずれにも該当しない場合、または判断が難しい場合は、ユーザーに確認を求めます。

## ステップ:

1.  **引数の解析**: ユーザーのリクエストから `source_file_path` を抽出します。
2.  **ファイルの存在確認**: `source_file_path` のファイルが存在することを確認します。
3.  **移動先の決定**: `source_file_path` の内容を読み込み、「自動的な移動先の決定」セクションのルールに基づいて最適な移動先ディレクトリを推測します。推測が不可能な場合、ユーザーに移動先を尋ねます。
4.  **ディレクトリの存在確認**: 決定された `destination_directory_path` のディレクトリが存在することを確認します。存在しない場合は作成を試みます。
5.  **ファイルの移動**: `source_file_path` のファイルを決定された `destination_directory_path` へ移動します。
6.  **移動の確認**: ファイルが正しく移動されたことを確認します。
