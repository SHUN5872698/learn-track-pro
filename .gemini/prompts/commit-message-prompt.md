# 絶対に使ってはいけない表現

❌ 「ます」「です」「します」
❌ 「〜された」「〜される」  
❌ 「これにより」「〜により」
❌ 「向上しました」「改善されました」

- 違反した場合は即座にやり直すこと。

# コミットメッセージ生成ルール

## 必須の出力形式

```
type: 変更内容の要約（30-50文字）

変更の全体的な説明を1行で記述（50-70文字）
- 具体的な変更点1（体言止め）
- 具体的な変更点2（体言止め）
- 具体的な変更点3（体言止め）
```


## typeの判定基準（厳密に適用）

| type | 使用条件 | 例 |
|------|---------|-----|
| feat | ユーザーに見える新機能・機能追加 | 新しいボタン、新しい画面、新しい機能 |
| fix | バグ・不具合の修正 | エラー修正、表示崩れ修正 |
| refactor | 内部構造の改善（動作は同じ） | コンポーネント化、重複削除、整理 |
| chore | 上記以外（設定、削除、依存関係） | 不要ファイル削除、パッケージ更新 |

## 文体ルール（必ず守る）

1. **タイトル**: 「〜を○○」の形式
   - ✅ 良い例: `汎用ボタンコンポーネントを導入`
   - ❌ 悪い例: `汎用ボタンコンポーネントの導入`

2. **説明文**: 「〜を○○」で終わる完全な文
   - ✅ 良い例: `アプリケーション全体で使用できる汎用ボタンコンポーネントを新規追加`

3. **箇条書き**: 体言止めまたは「〜を○○」
   - ✅ 良い例: `UIの一貫性と保守性を大幅に向上`
   - ❌ 悪い例: `UIの一貫性と保守性が向上しました`


## 実例テンプレート

### feat の例

```
feat: 汎用ボタンコンポーネントを導入

アプリケーション全体で使用できる多様なスタイルの汎用ボタンコンポーネントを新規追加
- プライマリ、セカンダリ、危険、アイコンなど多様なバリアント
- ボタンのスタイルと動作を一元管理
- UIの一貫性と保守性を大幅に向上
```

```
feat: ダッシュボードのモバイル対応を実装

ダッシュボードのレイアウトをモバイルデバイス向けに最適化し、ユーザー体験を向上
- LearningContentCardのセクション数と総学習時間の表示レイアウトを調整
- StatsOverviewのアイコンサイズを調整
- DashboardLayoutのヘッダーレイアウトを調整
- Dashboardのボタンとコンテンツセクションのレイアウトを調整
```

```
feat: 学習記録のフィルタリング機能を追加

学習記録一覧ページに日付範囲とカテゴリによるフィルタリング機能を実装
- 日付範囲ピッカーコンポーネントを導入
- カテゴリ選択UIを追加
- フィルタ条件をURLクエリパラメータで管理
- ユーザーの記録検索効率を大幅に改善
```

### fix の例

```
fix: ログイン時のCSRFトークンエラーを修正

Fortifyエンドポイントへのリクエスト時にカスタムAxiosインスタンスを使用していた問題を解決
- 認証エンドポイントで素のAxiosを使用するよう変更
- CSRFトークンの取得と送信フローを正常化
- ログイン失敗率を大幅に削減
```

```
fix: モバイル表示時のナビゲーションメニューの表示崩れを修正

画面幅が狭い場合にメニュー項目が重なる問題を解決
- ハンバーガーメニューの実装を追加
- ブレークポイントをmd（768px）に統一
- タッチ操作時の反応速度を改善
```

### refactor の例

```
refactor: 既存ボタンを汎用コンポーネントに置き換え

既存のボタン要素を汎用ボタンコンポーネントに統一的に置き換え
- モーダル、カード、フォーム、認証ページなどで適用
- コードの重複を削減し保守性を向上
- UIの統一化により一貫性を確保
```

```
refactor: エラーハンドリングロジックを共通化

各コンポーネントに散在していたエラーハンドリング処理をComposableに集約
- useErrorHandlerを新規作成
- API呼び出し時のエラー処理を統一
- コードの重複を70%削減
```

```
refactor: 定義したTailwind CSSコンポーネントクラスを各画面に適用

app.cssで定義した共通クラスを全Vueコンポーネントに適用し、スタイルを統一
- section-header、form-input-baseなどの共通クラスを適用
- インラインスタイルの記述を削減
- CSSの保守性を大幅に向上
```

### chore の例

```
chore: 未使用のテストファイルを削除

開発初期に作成した不要なテストファイルとモックデータを整理
- 使用されていないテストファイル5個を削除
- 古いモックデータディレクトリを削除
- プロジェクト構造を簡潔化
```

```
chore: ESLint設定を更新

Vue 3 Composition API向けのルールセットを追加し、コード品質基準を強化
- eslint-plugin-vueのバージョンを更新
- script setupタグ用のルールを追加
- 自動フォーマット時の一貫性を向上
```
